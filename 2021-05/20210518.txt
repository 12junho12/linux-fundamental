Cisco IOS

Cisco IOS Modes

 - User Exec Mode : 부팅 후 최초의 모드로 가장 기능의 제약이 많은 모드이다.
                            다른 장비로의 접속, 장비의 네트워크 연결 테스트(ping) 만 가능함
                            Router>

 - Privileged Mode : 네트워크 장비의 설정, 상태정보 등을 확인할 수 있고 설정을 저장할 수 있는 모드
                             Router#

 - Global Configuration Mode : 네트워크 장비의 전역 설정을 할 수 있는 모드
                              Router(config)#
 - Interface Mode : 네트워크 장비의 Interface를 설정할 수 있는 모드
                              Router(config-if)#

 - Line Interface Mode : Console, 원격접속 등의 Line Interface를 설정할 수 있는 모드
                              Router(config-line)#


Configuration 종류
 - running-config : 네트워크 장비에서 현재 동작하며 사용중인 구성
                          (RAM)
 - startup-config : 네트워크 장비에 영구적으로 저장되는 구성
                         (NVRAM)

설정 영구 저장
 Router# copy running-config startup-config


네트워크 장비 패스워드 설정
 - enable password
     - 네트워크 장비의 User Mode에서 Privileged Mode로 전환하기 위한 패스워드(기본적으로 평문으로 저장)
 - enable secret
     - 네트워크 장비의 User Mode에서 Privileged Mode로 전환하기 위한 패스워드(기본적으로 암호화되어 저장)
     - enable password, secret 둘다 설정된 경우 enable secret이 우선한다.


네트워크 장비 패스워드 재설정
1. 장비의 전원을 껐다가 켠다.
2. 부팅 중 [Ctrl]+[Pause Break] 로 ROMMON 모드로 진입한다.
3. Configuration Register 값을  0x2142 로  변경한다.
    ROMMON 1 > confreg 0x2142
4. ROMMON 모드에서 부팅을 진행한다.
    ROMMON 2 > boot
5. startup-config를 로드하지 않고 부팅이 된다.
6. 패스워드를 재설정한다.
    Router(config)# enable password cisco
7. Configuration Register 값을 0x2102  로  변경한다.
    Router(config)# config-register 0x2102
8. 네트워크 장비의 설정을 영구적으로 저장한다.
    Router# copy running-config startup-config
9. 네트워크 장비를 재부팅 해본다.
    Router# reload

Ch02. IP 주소 할당(IP Addressing)


Class A
Class B
Class C

특수 IP 주소
0.0.0.0  :  모든 네트워크
255.255.255.255 : 브로드캐스트
127.0.0.0 : Loopback

공인 IP 주소
 - 공인 네트워크 망(Internet)에서 사용될 수 있는 IP 주소

사설 IP 주소
 - 사설 네트워크 내에서만 사용할 수 있는 IP 주소

10.0.0.0 ~ 10.255.255.255
172.16.0.0 ~ 172.31.0.0
192.168.0.0 ~ 192.168.255.255


서브네팅(Subnetting)
 - 원래의 IP 네트워크를 더 작은 단위로 쪼개어 네트워크를 사용하는 기법

서브넷 마스크(Subnetmask)
 - IP 주소에서 Network ID와 Host ID를 구분하기 위한 마스크 값

Class A : 255.0.0.0
Class B : 255.255.0.0
Class C : 255.255.255.0
