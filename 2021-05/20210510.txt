게이트웨이(Gateway)
 - 서로 다른 네트워크를 연결하는 네트워크의 관문역할을 하는 장비
 - 다른 네트워크로 가고자 할 때 거치는 네트워크 장비

리피터(Repeater)
 - 전송매체를 통해 시그널 전송시 신호 감쇠, 손실 등이 발생하게 되어 전송매체별로 최대 도달 가능한 거리가 있음.
 - 전송매체를 통해 시그널 전송시 신호 감쇠가 되지 않도록 중간에 감쇠된 신호를 증폭해주는 네트워크 장비

라우터(Router)
 - 서로 다른 네트워크와 네트워크를 연결해주는 네트워크 장비
 - 목적지 네트워크로 도달하기 위한 경로를 계산하여 경로를 결정한다.


네트워크 토폴로지(Network Topology)
 - 호스트와 호스트가 네트워크로 연결된 연결 방식 또는 연결 상태

성형(Star Topology)
 - 중앙에 Controller 역할을 수행하는 네트워크 장비가 있고 이 장비를 중심으로 모든 호스트들이 연결됨.
 - 네트워크의 성능, 신뢰성 등은 Controller에 의해 결정됨.

 장점
   - 중앙집중식으로 관리하기 편하며 유지보수에 용이하다.
   - 링크 하나에 문제 발생시 문제된 링크만 통신이 되지 않고 네트워크 전체로 장애가 파급되지 않는다.

 단점
   - 중앙에 있는 네트워크 장비에 장애 발생시 네트워크 전체의 통신이 불가하다.


버스형(Bus Topology)
 - 네트워크 상의 모든 호스트 들이 중앙의 메인 케이블(Bus)를 통해 연결되는 네트워크 토폴로지
 - 각 호스트들은 탭을 이용하여 메인 버스와 연결하고 버스의 양 끝단에는 반드시 터미네이터를 연결해야한다.

  장점
    - 설치가 간편하고 구축비용이 매우 적게 든다.
    - 다른 토폴로지에 비해 소요되는 케이블의 양이 적다. 

  단점
    - 장비의 대수가 많아지면 네트워크 성능이 저하됨
    - 중앙의 메인 버스가 손상되면 전체 네트워크의 통신이 불가함
    - 재구성(재배치)가 어렵다.
    - 한 노드에서 데이터 전송시 다른노드에서 데이터를 전송하고 있는 경우 충돌이 발생하게되어 기다린 후 다시 재전송을 해야함


트리형(Tree Topology)
  - 성형(Star Topology)의 변형으로  모든 호스트를 중앙의 Controller에 연결하지 않는 토폴리지
  - 계층을 두어 상위 계층의 노드가 하위 계층의 노드를 관리하는 형태

  장점
    - 유지보수가 용이하고 네트워크의 확장이 용이하다.
    - 중앙의 Controller는 일반 호스트가 아닌 다른 허브/스위치 등을 연결하여 더 많은 호스트들을 연결할 수 있음
  단점
    - 중앙의 Controller에 네트워크 통신이 집중되어 병목현상이 발생할 수 있음


링형(Ring Topology)
 - 호스트들이 순차적으로 연결된 형태로 모든 호스트들이 하나의 링(고리)형태로 연결됨
 - 각 호스트들은 인접 호스트 2대(양 옆)와 연결하는 구조를 가짐
 - 링형 토폴로지는 링 구성에 따라 Single Ring, Double Ring으로 구분됨

  장점
    - 구조가 단순하여 구성이 용이하고 장애 발생시 복구시간이 빠름
    - 스타 토폴리지보다 케이블 소모가 덜함
    - 링형 토폴로지의 신호는 순환하게되며 일정시간 이내에 신호를 받지 못하면 장애발생사실 확인이 가능함

  단점
    - 제어가 복잡함
    - 새로운 장비 연결시 링을 해체 후 재구성이 필요함
    - 링 내의 장비에 장애 발생시 전체 네트워크 통신이 불가함

그물형(Mesh Topology)
 - 중앙에 Controller 없이 네트워크에 속하는 모든 호스트들이 1:1(Point-to-Point)로 연결되는 토폴로지
 - 각 호스트들은 두 호스트 간의 전용의 회선이 구성되고 매시 토폴로지 구성을 위한 링크의 갯수는  n(n-1)/2 개의 링크가 소요된다.
 - 네트워크 토폴로지 중 비용이 가장 많이 드는 토폴로지 구성 방식이며 구성이 매우 복잡하다.
 - 네트워크 안정성이 뛰어나므로 중요 네트워크에서 사용되는 구성 방식임
 - 매시 토폴로지에는 모든 호스트와 연결되는 Full Mesh와 일부 호스트와의 연결이 끊어져 있는 Partial Mesh Topology가 있음

 장점
   - 전용 링크를 사용하여 각각 원하는 데이터 전송이 가능하므로 회선 부하를 분산할 수 있음
   - 링크에 장애 발생시 대체 경로를 통한 우회 접근이 가능함
   - 전용 링크를 사용하여 데이터를 전송하므로 물리적으로 원하는 대상만 데이터를 받을 수 있어 보안 구성에 유리함

 단점
   - 비용이 많이 들고 유지보수가 복잡함
   - 링크가 많이 소요되므로 링크를 수용할 수 있는 충분한 공간이 확보되어야 함
