<Linux 파일시스템 계층 구조>

/bin  : 시스템 명령어 디렉터리
/boot  : 부팅 설정 파일 디렉터리
/dev  : 시스템의 장치 파일 파일 위치 디렉터리
/etc   : 시스템의 구성파일 위치 디렉터리
/home : 일반 사용자의 홈디렉터리가 위치 경로
/usr   : 패키지 설치시 공유 파일 등이 위치하는 경로
  /usr/bin : 시스템 명령어
  /usr/sbin : 시스템 관리 명령어
  /usr/lib   : C 라이브러리 등이 위치함
  /usr/share : 문서, 샘플 설정 파일 등
/mnt  : 디스크 장치 등을 연결(마운트)하기 위한 경로
/root  : root의 홈디렉터리
/run   : runtime 실행시 저장 데이터 저장을 위한 경로
/tmp  : 임시 파일을 저장하기 위한 경로
/var   : 가변 데이터(Web, FTP, Log)가 위치하는 경로

Ch07. 퍼미션의 이해 및 사용

퍼미션
 - 파일/디렉터리에 접근 가능한 권한
 - read, write, executable 권한을 조합하여 표현
 - 파일 소유자, 소유그룹, 기타 사용자에게 권한을 부여

 - $ chmod  [option]  PERM  FILE

 - 8진수

 - 심볼릭

[퍼미션 해석]
           파일                  디렉터리
read  파일 읽기              파일 목록 확인
write  파일 내용 수정       파일 생성 및 삭제
exec  파일 실행              디렉터리 이동(cd, 디렉터리 접근)


umask (기본 권한)
 - 파일 생성시 기본 권한을 지정하기 위해 배제할 권한을 지정하는 값

Ch08. 쉘 명령어 사용

쉘 메타 문자 : 쉘에서 특별한 의미를 가지는 메타 문자

(1) 경로 이름 메타 문자
 1) ~  : 현재 로그인 된 사용자의 홈 디렉터리 
      ~
      ~user
      ~+
      ~-


(2) 파일 이름 대체 메타 문자
  1) *  : 아무것도 없거나 하나 이상의 문자를 대체하는 메타문자
  2) ?  : 임의의 한 문자를 대체하는 메타문자
  3) []  : 정의된 문자 세트 내의 한 문자로 대체하는 메타문자


(3) 인용부호 메타 문자
  1) ' '  : 홑 따옴표 안에 있는 모든 메타문자를 일반 문자로 취급하는 인용부호 메타문자
  2) " " : 이중 따옴표 안에 있는  ' $ ', ' ` ', ' \ ' 이 세가지 메타문자를 제외한 모든 메타문자를 일반 문자로 취급하는 인용부호 메타문자


(4) 방향 재지정 메타 문자
