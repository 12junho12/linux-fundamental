오픈스택 노드
- 컨트롤러 : 관리 노드, HA구성 (Pacemaker)
	시스템 부하가 클 수 있음(CPU/RAM/네트워크)
- 컴퓨트 : 인스턴스를 실행 -> 사용자가 사용할 리소스
	(CPU/RAM/DISK 등의 리소스를 인스턴스에 할당)
- 네트워크 : 네트워크 기능(외부연결) => 네트워크 병목현상
- 스토리지 : Cinder/Swift 등의 스토리지 제공

프로젝트 - 하나의 작업 공간 (격리된 공간)
	각각 리소스 제한량 설정
	예전에는 tenant라는 용어도 사용

인스턴스 - 가상머신, 예전에는 Server라는 용어도 사용

이미지 - 인스턴스의 기본요소
	부팅 가능한 운영체제가 설치된 가상머신
네트워크 - 인스턴스의 기본요소
	인스턴스의 연결을 담당 (VM-VM /VM-외부)

플레이버 - 이름 f1, cpu: 1, ram: 2G, disk: 10G
이미지 - 이름: img1, 보호 활성화
	이름: img2, ram: 4G
네트워크 - 이름: net1, 서브넷이름: sub1
	서브넷대역: 192.168.0.0/24, dhcp 활성화
인스턴스 - 이름: vm1, 이미지: img1, 플레이버: f1, 네트워크: net1
	이름: vm2, 이미지: img2, 플레이버: m1.medium, 네트워크: net1

기본 인스턴스 - 플레이버 / 이미지 / 네트워크(내부)
	-> 외부에서 접속할 수 없음, 외부로 접근도 못함
퍼블릭 인스턴스 - 추가적으로 외부 네트워크와 라우터가 필요
	->여기까지만 하면 인스턴스에서 외부로만 가능
	+유동IP(floating IP)와 보안그룹(방화벽)을 설정
	=>외부에서도 접근  가능

인스턴스 포함해서 네트워크 / 라우터 / 이미지 / 플레이버 들을  전체 제거
직접 외부(8.8.8.8)로 핑이 전달되는 인스턴스 생성
이미지: cirros -> 이름만 지정
플레이버: 10G / 2048M / 1core + eph 디스크 1G 추가
내부 네트워크 : 192.168.100.0/24 대역으로 생성, dhcp O
외부 네트워크 : flat, extnet, 10.0.2.0/24, 10.0.2.1 , dhcp X
라우터 연결 후 ping 테스트 및 디스크 확인
