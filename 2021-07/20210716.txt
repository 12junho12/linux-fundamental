Neutron - 네트워크 관리 서비스
ML2
-  Modular Layer2(2계층)
- Havana 버전에서 소개
- 네트워크 구성 방식 (OVS/OVN) 중에서 OVS에서는 계층 구분
- 타입 드라이버: Local, Flat, Vlan, VXLAN, GRE, Geneve
- Local: 단일 노드에서 테스트용으로 사용
- Flat: 다중 노드에서 사용
-  Vlan: 스위치에서 제공하는  Vlan 기능 사용( 4095개)
- GRE: 터널링 기술
- VXLAN: VLAN의 확장버전

보안그룹
- 인스턴스의 패킷 제어를 위해 사용
- 인스턴스에 직접 연결되는 Linux-bridge에서 사용
- 설정을 변경하는 즉시 적용
- 명령어 설정시에는 포트 번호로만 지정 가능

내부에서 인스턴스간에 통신을 확인
1. 내부 네트워크 하나에 인스턴스를 2개 연결(172.168.0.0/24)
	Cirros 이미지로 생성, 보안그룹은 둘다 default만
	-> 2개의 인스턴스가 같은 네트워크에 연결
	보안그룹이 같은 default이므로 IP주소도 확인가능 / 규칙에 따라 패킷 허용
2. 내부 네트워크 하나 더 생성, 인스턴스 1개 추가
	(172.160.100.0/24, 보안그룹 default)
	-> 네트워크가 분리( IP 주소를 확인할 수 없음)
3. 172.160.100.0/24 네트워크에 인스턴스 추가
	단, 보안그룹은 새로 만들어서 설정
	보안그룹 규칙: 모든 IP 및 프로토콜에 대해 허용
	-> 1,2 인스턴스는 네트워크가 다름(IP주소 확인X)
	3 인스턴스는 IP 주소는 확인. 보안그룹 규칙 X
	대신 3->4는 규칙이 허용

네임스페이스
- 네트워크 / PID / 장치 ...
- 격리
- ip netns 명령어로 확인 / 관리

Cinder - 블록스토리지 서비스
-LVM, NFS, GlusterFS, Ceph 등	
- 스토리지 노드에 별도의 스토리지(OS제외) 필요
- 인스턴스에 연결해서 사용
- 인스턴스의 데이터 저장 용도


Swift - 오브젝트 스토리지 서비스
- 다양한 종류의 파일(데이터)을 저장
- 인스턴스와 무관하게 사용하는 단순 저장소
- Glance 서비스의 백앤드 스토리지로 사용 (이미지를 저장)
- Cinder 서비스의 백업을 위한 용도
- 인스턴스의 백업을 위한 용도

Heat - 오케스트레이션(조율)
- 리소스를 생성 / 관리 하는 서비스(템플릿)
- cloudform , HOT(heat orchestration template) 등을 사용
- 리소스를 정적 /  동적 관리 가능
- 템플릿으로 스택을 생성
- 스택이란 템플릿에 정의된 리소스들의 집합
- 여러 리소스를 동시 관리

기존작업: dashboard/command -> keystone -> 서비스
템플릿 사용: dashboard/command -> keystone -> Heat -> 서비스
